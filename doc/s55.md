## Scribe là gì?

Scribe tự động đọc code (routes, controllers, FormRequests, PHPDoc) và sinh ra:
- **Trang web docs** đẹp tại `/docs` — có thể test API trực tiếp trên trình duyệt
- **Postman collection** — import 1 click vào Postman
- **OpenAPI spec** — dùng cho Swagger hoặc tool khác

Không cần viết docs tay. Chạy 1 lệnh là xong.

---

## Cấu hình hiện tại (`config/scribe.php`)

| Setting | Giá trị | Ý nghĩa |
|---|---|---|
| `title` | YourShop API Documentation | Tiêu đề trang docs |
| `type` | laravel | Render qua Blade view |
| `routes.prefixes` | `api/v1/*` | Chỉ scan routes `/api/v1/` |
| `auth.enabled` | true | Bật auth |
| `auth.default` | true | Mặc định tất cả endpoints cần auth |
| `auth.in` | bearer | Dùng Bearer token |
| `postman.enabled` | true | Sinh Postman collection |
| `openapi.enabled` | true | Sinh OpenAPI spec |

## Cách dùng

### 1. Xem docs trên trình duyệt

```bash
php artisan serve
# Mở http://localhost:8000/docs
```

> Chỉ hoạt động trên `local` và `staging`. Production không expose.

---

### 2. Regenerate docs (khi thêm endpoint mới)

```bash
php artisan scribe:generate
```

Chạy lệnh này mỗi khi:
- Thêm controller/endpoint mới
- Sửa FormRequest (thêm/bớt field)
- Sửa PHPDoc

---

### 3. Import Postman collection

1. Mở Postman
2. **Import** → chọn file `storage/app/private/scribe/collection.json`
3. Tất cả endpoints có sẵn với ví dụ request

---

### 4. Dùng OpenAPI spec

```
storage/app/private/scribe/openapi.yaml
```

Import vào Swagger UI, Insomnia, hoặc bất kỳ tool nào hỗ trợ OpenAPI 3.0.

## Cách thêm PHPDoc cho endpoint mới

Scribe đọc các annotation trong PHPDoc. Ví dụ:

```php
/**
 * Tên endpoint hiển thị trên docs
 *
 * Mô tả chi tiết hơn nếu cần.
 *
 * @unauthenticated              ← endpoint này không cần auth (override default)
 * @authenticated                ← endpoint này cần auth (default nên thường bỏ qua)
 *
 * @urlParam slug string required Slug của sản phẩm. Example: iphone-15
 *
 * @queryParam page int Trang. Default: 1. Example: 1
 * @queryParam per_page int Số item/trang. Default: 15. Example: 15
 * @queryParam category string Lọc theo slug danh mục. Example: smartphones
 *
 * @bodyParam name string required Tên sản phẩm. Example: iPhone 15
 * @bodyParam price number required Giá. Example: 22990000
 *
 * @response 200 scenario="Success" {"success":true,"data":{...}}
 * @response 404 scenario="Not found" {"success":false,"message":"Not found"}
 * @response 422 scenario="Validation error" {"success":false,"errors":{...}}
 */
public function show(string $slug): JsonResponse
```

### Annotation hay dùng

| Annotation | Dùng khi |
|---|---|
| `@unauthenticated` | Endpoint public, không cần token |
| `@urlParam` | Tham số trên URL (`:slug`, `:id`) |
| `@queryParam` | Query string (`?page=1`) |
| `@bodyParam` | Body của POST/PUT request |
| `@response` | Ví dụ response (có thể thêm nhiều scenario) |

## Scribe đọc FormRequest tự động

Nếu controller dùng FormRequest, Scribe tự đọc `rules()` để sinh body params — không cần `@bodyParam` thủ công.

```php
// StoreProductRequest.php
public function rules(): array
{
    return [
        'name'  => ['required', 'string', 'max:255'],
        'price' => ['required', 'numeric', 'min:0'],
        'slug'  => ['required', 'string', 'unique:products'],
    ];
}
```

Scribe sẽ tự sinh docs cho 3 fields này. Tuy nhiên nếu muốn thêm mô tả, dùng `bodyParameters()`:

```php
public function bodyParameters(): array
{
    return [
        'name'  => ['description' => 'Tên sản phẩm', 'example' => 'iPhone 15'],
        'price' => ['description' => 'Giá bán (VND)', 'example' => 22990000],
    ];
}
```

## Lưu ý

- Route `/docs` chỉ mở trên `local` và `staging` — xem `routes/web.php`
- Scribe là `--dev` dependency — không có trong production build
- Mỗi lần thêm endpoint mới nhớ chạy `php artisan scribe:generate` để cập nhật docs
